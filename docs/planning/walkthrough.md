# 游戏机制还原：卡牌驱动竞速 (Heat-Style)

我们已经成功将游戏机制从“骰子+卡牌”混合模式还原为更硬核的**纯卡牌驱动模式**，更接近《Heat: Pedal to the Metal》桌游的原始玩法。

## 主要变更

### 1. 移除掷骰子 (Dice Removal)
- **服务端**：禁用了 `player-move` socket 事件，防止任何非卡牌驱动的位移。
- **客户端**：从 `CardHand` 组件中彻底移除了“掷骰子”按钮及相关动画逻辑。
- **单机模式**：AI 不再尝试进行随机骰子移动。

### 2. 档位与卡牌深度关联 (Gear-Card Synergy)
- 玩家的**档位 (Gear)** 决定了本回合的**行动点 (Action Points)**。
- 每一张移动卡消耗 1 个行动点。
- 玩家必须通过合理管理手牌和档位来控制车速，从而在不爆缸的前提下最快过弯。

### 3. 手牌管理优化 (Hand Refill Optimization)
- **手牌上限提升**：为了支持高档位（如 4 档以上）的大量出牌，我们将手牌上限从 5 张提升至 **7 张**。
- **回合结束补满**：不再是每回合只抽 1 张牌，而是还原了桌游机制——在回合结束时将手牌**补满至 7 张**，确保下回合有足够的策略空间。

### 4. UI 与规则更新
- **GameRulesModal**：重写了规则说明，删除了随机移动内容，强调了档位和卡牌的关系。
- **CardHand**：重新排版了操作区域，使“结束回合”按钮在纯卡牌模式下更醒目。

## 验证结论
- **移动逻辑**：已验证 `playCard` 依然能正确触发 `_movePlayer` 及其内部的过弯超速（Heat）、尾流（Slipstream）和爆缸逻辑。
- **AI 行为**：AI 现在会根据当前档位尝试打出多张移动卡，直至行动点耗尽，行为更具竞争性。
- **同步性**：服务端和客户端的 `HAND_LIMIT` 已同步为 7，避免了状态不一致导致的错误。
