# A. 卡牌系统完整规格

## A.1 卡牌设计原则

### A.1.1 卡牌稀有度与数量分布

| 稀有度 | 定义 | 典型数量/组 | 出现概率 |
|--------|------|------------|----------|
| 普通（C） | 基础移动，+1至+3步 | 15-20张/组 | 55% |
| 稀有（R） | 特殊效果，移动+2至+4步 | 6-8张/组 | 25% |
| 史诗（S） | 强力效果，影响多个维度 | 3-5张/组 | 13% |
| 传说（L） | 改变局势，决定胜负 | 1-2张/组 | 7% |

### A.1.2 能量期望值（EVA）系统

每张卡的**能量期望值（Expected Value of Action, EVA）** 用于平衡卡牌强度：

```
EVA = 基础移动值 × 使用率因子 × 风险系数

使用率因子：
  移动牌 = 1.0
  攻击牌 = 0.7（目标不一定在场）
  防御牌 = 0.5（防御时机不确定）
  特殊牌 = 0.6（条件触发）

风险系数：
  无风险 = 1.0
  轻微风险（如对自己有负作用）= 0.85
  中等风险（如过热风险）= 0.7
  高风险（如爆缸可能）= 0.5
```

**标准EVA参考**：
- EVA ≈ 2.0：普通牌（合理）
- EVA ≈ 3.0：稀有牌（强力）
- EVA ≈ 4.5：史诗牌（优秀）
- EVA ≈ 6.0：传说牌（罕见，决定性）

# B. 热量平衡参数

## B.1 核心数值设计

### B.1.1 热量池参数

| 参数 | 数值 | 设计理由 |
|------|------|----------|
| **热量池上限** | 3张 | 低上限→频繁需要冷却决策，高压力 |
| 热量卡热值 | 均为1张 | 简化计算，避免复合热量 |
| 过弯超速惩罚 | 超出限速值 = 需生成热量卡数 | 直接对应，风险明确 |
| 热量惩罚倍率（GEAR） | 见下表 | 档位越高惩罚越重 |

#### 档位热量惩罚倍率表（GEAR模式）

| 档位 | 必须出牌数 | 热量惩罚倍率 | 示例（超速2点） |
|------|-----------|------------|----------------|
| 1档 | 1张 | ×0 | 不产生热量（1档不超速） |
| 2档 | 2张 | ×1 | 超速2点 → 2张热量卡 |
| 3档 | 3张 | ×1.5→1 | 超速1点 → 1张（向下取整） |
| 4档 | 4张 | ×2 | 超速1点 → 2张热量卡 |
| 5档 | 5张 | ×2.5→2 | 超速1点 → 2张热量卡 |
| 6档 | 6张 | ×3 | 超速1点 → 3张热量卡 |

**注**：若玩家手牌不足所需热量卡数，则用全部手牌填充，剩余部分由弃牌堆抽取（等同于强制洗牌）

### B.1.2 过弯热量结算完整算法

```
INPUT: playerState, currentTile, speed
OUTPUT: heatGenerated, moveResult

1. 确定有效速度
   effectiveSpeed = speed
   // 以下效果可降低有效速度
   IF 打出"安全驾驶"牌: effectiveSpeed -= 1
   IF 打出"引擎调校"牌: effectiveSpeed -= 1 (本回合热量惩罚-1)
   IF 尾流效果生效: effectiveSpeed += 2 (尾流移动不计入)
   
2. 确定有效限速
   effectiveLimit = currentTile.speedLimit
   IF TIRE开启 && tireTemp == 'cold': effectiveLimit -= 1
   IF 打出"弯道减速"牌: effectiveLimit += 1
   IF LEAN开启 && 宣告极限压弯: effectiveLimit += 2
   
3. 热量结算
   IF effectiveSpeed > effectiveLimit:
     overSpeed = effectiveSpeed - effectiveLimit
     heatGenerated = overSpeed * gearMultiplier[currentGear]
     
     // 从手牌转移热量卡
     heatCardsToMove = MIN(heatGenerated, 手牌中非热力卡数量)
     moveRemainingHeatCards = heatGenerated - heatCardsToMove
     
     // 热力卡只能从手牌转移，不足则从弃牌堆补
     heatCards = 从手牌选择heatCardsToMove张 → 热量池
     IF moveRemainingHeatCards > 0:
       从弃牌堆抽取moveRemainingHeatCards张 → 热量池
       
4. 爆缸判定（CRASH模块）
   IF heatPool.size >= heatCapacity && heatGenerated > 0:
     triggerOverheat()
```

### B.1.3 冷却效率表

## B.2 平衡性测试用例

### B.2.1 典型场景分析

**场景1：2档安全过弯（无热量）**
```
初始状态：2档，手牌5张，热量池0/3
行动：打出1张+1步基础牌，移动力=2
经过弯道：限速4，速度2
结果：2 < 4，无热量产生
→ 热量池：0/3，安全
```

**场景2：4档激进过弯（中等热量）**
```
初始状态：4档，手牌6张，热量池1/3
行动：打出3张+1+2+1，移动力=4
经过弯道：限速4，速度4（有效速度=4）
结果：4 = 4，无热量产生（恰好等于不超速）
→ 热量池：1/3，维持
```

**场景3：4档超速过弯（危险）**
```
初始状态：4档，手牌6张，热量池2/3
行动：打出4张+2+2+1+1，移动力=6
经过弯道：限速4，速度6
结果：6 > 4，超速2点 × 2倍惩罚 = 4张热量卡
→ 热量池：满 → 爆缸！
→ 惩罚：停止+降至1档+热量池清空
```

**场景4：爆缸后的恢复曲线**
```
爆缸后状态：1档，热量池0/3，手牌3张
第1回合：1档起步，只能出1张，移动力约1-2
第2回合：可换2档，开始恢复
第3回合：可换3档，恢复正常节奏
→ 爆缸惩罚持续约2-3回合
```

### B.2.2 数值平衡目标

| 指标 | 目标范围 | 测试基准 |
|------|----------|----------|
| 平均热量池持有量 | 0.5-1.5张 | 模拟1000局 |

## C.3 各状态详细说明

### C.3.1 LOBBY → WAITING → STARTING

```
LOBBY:
  - 房主设置游戏模式（模块开关）
  - 其他玩家申请加入
  - 房主可踢人、设置旁观模式
  - 玩家准备/取消准备（≥2人才能开始）
  
WAITING:
  - 等待所有玩家准备
  - 房主可随时点击"开始游戏"
  - 倒计时：30秒后自动开始（可取消）
  
STARTING:
  - 3-2-1倒计时（每秒广播）
  - 广播游戏配置（模块开关、赛道、目标圈数）
  - 初始化所有玩家状态
  - 洗牌 → 每人发初始手牌
  - 第1回合开始
```

### C.3.2 TURN_GEAR_CHANGE（GEAR模式）

```
进入条件：modules.gearSystem == true

① 广播当前档位信息给该玩家
② 玩家选择：升档 / 降档 / 保持
③ 规则检验：
   IF 升2档以上 && heatPool > 0:
     从热量池移除1张 → 弃牌堆（跳档税）
   ELSE IF 升2档以上 && heatPool == 0:
     拒绝 → 提示"热量池为空，无法跳档"
   ELSE:
     允许
④ 广播档位变更
⑤ 超时（15秒）：自动保持当前档位
```

### C.3.3 TURN_CARD_SELECTION

```
进入条件：任何模式

CORE（非GEAR）：
  ① 掷骰结果=本回合可用步数
  ② 玩家从手牌选择打出哪些加速牌（0-N张）
  ③ 广播出牌内容
  
GEAR模式：
  ① 当前档位=必须打出的最少牌数
  ② 玩家选择 requiredCards 张牌（可多选1张作为弃牌）
  ③ 立即结算总移动力
  ④ 超时（20秒）：自动打出档位数量最强的牌
  
无行动牌可用：
  → 跳过出牌，本回合仅使用基础步数/档位1档
```

### C.3.4 TURN_MOVE

```
① 计算总移动力
   baseMove = 掷骰结果 或 档位出牌总和
   modifiers = 格子效果加减 + 尾流加成
   
② 执行移动
   FOR 每1格:
     - 更新位置
     - 检查落点格子效果（立即触发）
     - 若格子效果导致停止（事故区）：立即中断移动
     
③ 格子效果触发优先级：
   1. 终点线（圈数+1）→ 检查胜利条件
   2. 事故区 → 停止移动，标记跳过下回合
   3. 维修站 → 触发冷却（若有）
   4. 氮气区 → 追加步数，继续移动
   5. DRS区 → 标记尾随状态
```

### C.3.5 TURN_CORNER（HEAT模块）

```
进入条件：modules.heatSystem == true && 移动路径跨越弯道

① 识别所有经过的弯道（可能有连续弯道）
② FOR 每个弯道:
   a. 获取有效速度（见B.2.3算法）
   b. 获取有效限速（基础限速+冷胎-1）
   c. 计算超速差值
   d. 生成热量卡并转移至热量池
   e. 爆缸判定：
      IF heatPool > heatCapacity:
        执行爆缸惩罚流程（见CRASH模块）
        中断后续弯道结算
   f. 广播热量变化给该玩家
```

### C.3.6 TURN_SLIPSTREAM（DRAFT模块）

```
进入条件：modules.slipstreamSystem == true

① 移动完成后，检查是否紧跟前方车辆
   "紧跟"定义：
     - 在同一赛道位置的正后方（同格子索引-1）
     - 或在相邻前方车辆的相邻格子

② IF 满足条件:
   a. 触发尾流效果
   b. 广播尾流触发信息
   c. 执行尾流移动（+2步，不触发格子效果）
   d. 可链式触发（尾流后再尾流，最多2次）

③ SPLIT模块：尾流超车判定
   IF modules.splitSystem && 尾流后超越≥2辆车:
     奖励：将1张热量卡移回手牌
```

### C.3.7 TURN_COMBAT（PVP模块）

```
进入条件：modules.combatSystem == true

① 攻击牌组广播
   a. 每个玩家检查是否有攻击牌可对当前场上目标使用
   b. 攻击牌按优先级排序（近>远，同距离按卡牌稀有度）
   
② 响应窗口开启（5秒）
   FOR 每个被攻击目标:
     - 广播攻击信息 + 防御窗口
     - 目标选择：打出防御牌 / 跳过
     - 响应超时 → 无防御
     
③ 结算层
   a. 护盾 → 攻击无效
   b. 紧急规避 → 减少1步惩罚
   c. 黑客反制 → 效果反弹给攻击者
   d. 无响应 → 攻击正常生效
   
④ 负面状态应用
   路障 → 添加debuff，1回合后自动清除
   电磁脉冲 → 添加debuff，跳过下回合出牌阶段
   漏油 → 添加地板标记，立即对经过玩家生效
```

### C.3.8 TURN_DISCARD

```
① 检查手牌数量是否超限
   handLimit:
     模式A/C: 5张
     模式B/D: 7张（热力卡不占手牌数）
     
② IF 手牌 <= handLimit:
   无需弃牌 → 进入TURN_END
   
③ IF 手牌 > handLimit:
   a. 广播弃牌窗口（10秒）
   b. 玩家选择超出的牌 → 弃牌堆
   c. 超时：自动选择点数最低的牌弃置
```

## C.4 边界情况处理

### C.4.1 断线处理

```
玩家断线：
  ① 暂停该玩家计时器（15秒等待重连）
  ② 其他玩家正常继续游戏
  ③ 重连后：
     - 发送完整游戏状态快照
     - 恢复玩家操作能力
  ④ 超时（30秒）:
     - 标记为"旁观者"
     - 移除该玩家车辆（留在原地）
     - 游戏继续（其他玩家可继续）
```

### C.4.2 牌库耗尽

```
① 抽牌时牌库为空：
   → 将弃牌堆洗牌 → 新的牌库
   → 继续抽牌
   
② 弃牌堆也为空：
   → 异常状态（正常不会发生）
   → 系统生成临时基础移动牌代替
   
③ 洗牌时游戏已结束：
   → 不再洗牌，抽牌视为跳过
```

### C.4.3 多人同时到达终点

```
判定顺序（优先级）：
  1. 先触发终点线的玩家获胜
  2. 同格判定：已完成圈数多者胜
  3. 圈数相同：格子编号大者胜（更接近终点）
  4. 完全相同：视为并列
```

### C.4.4 无效目标处理（PVP）

```
攻击牌目标不存在（玩家离开/超出范围）：
  → 攻击牌失效，不消耗（退回手牌）
  
防御牌打出但无攻击：
  → 防御牌失效，直接弃置
  
漏油陷阱目标已离场：
  → 标记不放置
```

# D. 胜负规则详述

## D.1 胜利条件

### D.1.1 标准胜利（圈数制）

**适用：所有模式**

```
完成条件：率先完成 targetLaps 圈（默认3圈）

胜利判定时机：
  ① 玩家移动后经过/停在终点线时
  ② 检查 lap + 1 >= targetLaps
  ③ 若满足 → 标记该玩家为"已完成"
  
一旦有玩家完成：
  → 广播"第一名完成"消息
  → 游戏继续（其他玩家继续比赛）
  → 直至所有玩家完成或放弃
```

### D.1.2 提前胜利条件

| 条件 | 触发场景 | 结果 |
|------|----------|------|
| **率先冲线** | 某玩家完成目标圈数 | 该玩家立即获胜（第一名） |
| **追及胜利** | 某玩家追上终点线的领先者 | 追上者获胜（尾流超越终点） |
| **最后一名退出** | 最后一名之外的玩家全部完成 | 最后一名获得最后一名次，游戏结束 |

## D.2 排名规则

### D.2.1 排名优先级

```
排名规则（按优先级降序）：
  ① 完成状态：已完成的玩家 > 未完成的玩家
  ② 完成圈数：已完成的圈数越多排名越高
  ③ 赛道位置：格子编号越大（越接近终点）排名越高
  ④ 同格处理：格子编号相同 → 出牌点数高者排前
```

### D.2.2 排名示例

```
目标圈数：3圈，当前第3圈进行中

玩家A：已完成3圈，在格10位置 → 第1名
玩家B：已完成3圈，在格5位置  → 第2名
玩家C：已完成2圈，在格34位置 → 第3名（虽在终点线但未完成）
玩家D：已完成2圈，在格20位置 → 第4名
```

## D.3 游戏结束流程

```
游戏结束触发：
  ① 所有玩家完成目标圈数
  ② 房主手动结束游戏
  ③ 任意玩家请求退出（≥2人离开则游戏结束）
  ④ 达到最大回合数（200回合强制结束）

结束流程：
  1. 停止所有计时器
  2. 计算最终排名
  3. 广播游戏结束面板
  4. 展示：排名、各玩家数据（里程/热力卡/攻击数等）
  5. 提供"再来一局"（保持房间）/ "返回大厅"选项
```

## D.4 积分系统（可选）

### D.4.1 单局积分计算

```
完赛排名积分：
  第1名：10分
  第2名：7分
  第3名：4分
  第4名：1分

额外加分（可选）：
  完成全部圈数：+2分
  零热量爆缸：+3分（技术奖励）
  击败第1名并获胜：+5分（逆袭奖励）
  触发尾流链式≥3次：+1分
```

### D.4.2 赛季积分（扩展功能）

```
赛季周期：每月重置

累计积分排行榜：
  月榜前3名 → 获得专属皮肤/称号
  赛季MVP → 稀有赛车皮肤
  
成就系统：
  - "完美一圈"：单圈无热量积累
  - "热力大师"：单局热量管理0爆缸
  - "弯道之王"：单局过弯最多且无惩罚
  - "尾流之王"：单局触发尾流≥10次
```

## D.5 特殊情况

### D.5.1 卡牌耗尽的游戏

```
牌库+弃牌堆全部耗尽：
→ 游戏不结束（改为无限牌库模式）
→ 每回合自动补充1张基础移动牌给每个玩家
→ 热力卡正常生成/冷却
→ PVP卡牌正常消耗
```

### D.5.2 超长游戏处理

```
最大回合数：200回合（防止无限游戏）

若达到最大回合数：
  → 强制结束游戏
  → 按当前排名计算积分
  → 广播"超时结束"消息
```

### D.5.3 平局处理

```
严格相同情况（几乎不可能）：
  ① 同一回合内2名玩家同时完成最后一圈
  ② 同格子同圈数
  
处理方案：
  → 并列排名（分数平均分配）
  → 例如：第1/2名并列 → 各得8.5分
  → 下一名直接排第3（无第2名）
```

```python
def ai_heat_decision(player, gameState):
    risk = len(player.heat_pool) / player.heat_capacity
    
    if risk >= 0.8:
        return 'EVACUATE'   # 全力散热
    elif risk >= 0.5:
        return 'CAUTIOUS'   # 谨慎
    else:
        return 'AGGRESSIVE'  # 激进
    
def ai_route_choice(player, corners, gear, hand):
    # 计算各路线热量风险期望值
    risks = {}
    for route in ['inner', 'outer']:
        risks[route] = sum(
            max(0, expected_speed(gear, hand) - c.speed_limit) 
            * GEAR_MULT[gear] * c.prob
            for c in corners_on_route(route)
        )
    return min(risks, key=risks.get)
```

### F.2.2 换档AI

```python
def ai_gear_decision(player, gameState):
    heat = len(player.heat_pool)
    ahead = get_tiles_ahead(player.position, n=6)
    corners = sum(1 for t in ahead if t.type == 'curve')
    nitros = sum(1 for t in ahead if t.type == 'nitro')
    
    if heat >= 3: return max(1, player.gear - 1)  # 强制降档
    if ahead_has_pit(): return min(player.gear + 1, 6)
    if heat >= 2: return player.gear if corners < 2 else player.gear - 1
    return min(player.gear + 1, 5) if nitros >= 2 else player.gear
```

### F.2.3 PVP AI

```python
def ai_attack_decision(player, gameState):
    attacks, defense = [], None
    for card in player.hand:
        if card.type == 'attack':
            for target in valid_targets(card, gameState):
                score = evaluate_target(card, target, player)
                if score > 20: attacks.append((card, target, score))
    
    for atk in gameState.incoming_attacks:
        for df in player.hand:
            if df.type == 'defense' and effect(df, atk) > 0:
                defense = df; break
    return attacks, defense
```

# H. 完整数据结构

```typescript
// 枚举
enum GameState { LOBBY, WAITING, STARTING, PLAYING, FINISHED }
enum TurnState { IDLE, CHOOSE_GEAR, GEAR_CHANGE, CARD, DICE, MOVE,
                  CORNER, SLIPSTREAM, TILE_EFFECT, COMBAT, DISCARD, END }
enum TileType { STRAIGHT, CURVE, NITRO, PIT, ACCIDENT, FORK_ENTRY,
                 FORK_INNER, FORK_OUTER, FORK_MERGE, START_FINISH }
enum CardType { MOVE, HEAT, ATTACK, DEFENSE, SPECIAL }

// 卡牌
interface Card {
  id: string;
  name: string;
  type: CardType;
  rarity: 'N' | 'R' | 'S' | 'L';
  moveValue?: number;       // 移动牌：+N步
  effect?: CardEffect;     // 特殊效果
  description: string;
  isPlayable: boolean;     // 当前是否可打出
}

// 热量（HEAT模块）
interface HeatState {
  pool: Card[];            // 热量池中的卡（均为热力卡）
  capacity: number;        // 容量上限，默认3
  isOverheat: boolean;     // 是否爆缸
}

// 档位（GEAR模块）
interface GearState {
  current: number;         // 当前档位 1-6
  canShiftFree: boolean;   // 是否可免费换档
}

// 轮胎（TIRE模块）
interface TireState {
  temp: 'cold' | 'warm';
  lastPitLap: number;      // 上次经过维修站的圈数
}

// 玩家
interface Player {
  id: string; name: string; color: string;
  position: number; lap: number; lapCount: number;
  hand: Card[]; handLimit: number;
  deck: Card[]; discardPile: Card[];
  gear: GearState;          // GEAR模块
  heat: HeatState;          // HEAT模块
  tire: TireState;          // TIRE模块
  bodyWeightMarkers: number; // WEIGHT模块，默认3
  debuffs: Debuff[];
  shieldActive: boolean;    // DEFEND模块
  isFinished: boolean; finishRank?: number;
  aiLevel?: 'easy' | 'normal' | 'hard' | 'expert';
}

// 格子
interface Tile {
  id: number; type: TileType; name: string; position: number;
  speedLimit?: number;       // 弯道限速，1-6
  isCheckpoint: boolean;
  effect: TileEffect;
  visual: { color: string; glow?: string; icon?: string };
}

// 赛道
interface Track {
  id: string; name: string; tiles: Tile[];
  totalTiles: number; difficulty: 1|2|3|4|5;
  theme: string;
}

// 房间配置
interface RoomConfig {
  modules: GameModules;
  targetLaps: 2|3|5;
  maxPlayers: 2|3|4;
  aiCount: 0|1|2|3;       // AI数量
  aiLevel: 'easy'|'normal'|'hard'|'expert';
  trackId: string;
}

// 游戏状态快照
interface GameSnapshot {
  state: GameState; turn: number;
  currentPlayerId: string;
  turnState: TurnState;
  players: Player[];
  activeMarkers: Marker[];
  turnTimeLeft: number;
}
```
